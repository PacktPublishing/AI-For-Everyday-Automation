# First Drafts in 15 Minutes

Generate polished business documents using a four-step connected chain.

## The Four-Step Chain

### Step 1: Connected Extraction
Gather inputs from Drive (meeting packets, prior reports, research briefs), task tool (completed/blocked tasks), Gmail (relevant threads), and user-provided content. Extract and organise. Flag gaps as [NEEDS INPUT].

### Step 2: Outline
Propose section-by-section outline showing heading, content, and which sources feed in. Wait for user approval. Adjust if redirected.

### Step 3: Draft
Write the full draft from the approved outline. CRITICAL: Only use content from extracted sources. NEVER invent facts. Mark gaps as [NEEDS INPUT]. Lead sections with key takeaways. Apply user's custom instructions.

### Step 4: Polish and Deliver
Tighten, strengthen opening, smooth transitions, add clear closing. Then: save to Drive, email via Gmail. Present final version for approval before sending.

## Pre-built Chains

### Weekly Status Report
Gather: Drive meeting packets + task tool data + relevant emails. Produce: overall status, accomplishments, in progress, blockers, next week. Under 400 words. Save and email.

### Project Proposal
Gather: research brief + meeting packets + budget document from Drive. Produce: executive summary, problem, solution, outcomes, costs, risks, next steps. 2-3 pages. Lead with recommendation.

### Client-Facing Summary
Gather: meeting packet + client emails + task deliverables. Produce: discussed, outcomes, next steps, open items. Safety check: no internal commentary, verified commitments, client-appropriate tone.

## Failure Fixes
- **Invented content:** Check against extraction. Delete anything not in inputs.
- **Wrong structure:** Redirect at Step 2.
- **Wrong tone:** Targeted follow-up describing changes.
- **Too long:** Specify cut percentage and what to keep vs cut.

## Template Library
Save proven chains as Google Docs in Templates folder. Future use: "Read my template from Drive and run the chain."
